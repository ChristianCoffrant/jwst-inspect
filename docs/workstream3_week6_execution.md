# Workstream 3 Week 6 Execution

## Scope

Week 6 freezes the Team 3 autonomous-inspection evaluation contract at version 0.2.0 and adds a one-command dev evaluation suite. The suite runs the existing local thin-slice, scripted sunshield survey, and learned state-baseline evaluators, then writes one deterministic metrics table for handoff.

No official Vast.ai GPU result is claimed by this week. The Vast X090 template now declares the metadata and storage-sync requirements that must be satisfied before any future official GPU run is accepted.

## Iterations

1. Freeze the evaluation contract.
   - Version the episode and metrics schemas at 0.2.0.
   - Declare the dev-suite tasks: `approach_hold_standoff` and `sunshield_survey`.
   - Declare the official dev-suite baselines: `scripted_baseline` and `learned_state_bc_v0_1`.
   - Add sensor-noise, latency, and actuation-delay hooks with no-op defaults.

2. Validate the guardrails.
   - Reject metric-weight drift from runtime scoring constants.
   - Reject unknown profile names.
   - Require official-run metadata for future Vast runs.
   - Keep final held-out seeds owned by the integration council.

3. Run the suite from config.
   - Command: `python scripts/run_dev_evaluation_suite.py --config configs/experiments/dev_evaluation_suite_v0_2.yaml --output-dir runs/dev_evaluation_suite`
   - Outputs are generated under `runs/dev_evaluation_suite` and should not be committed.

## Guardrails

- Metric weights may not change after freeze without integration-council approval and ablation evidence.
- Metrics tables must be generated by scripts, not manually edited.
- Official GPU runs require registry metadata and synced artifacts.
- Unknown latency, sensor-noise, or actuation-delay profiles fail validation.
- Default no-op profile hooks must preserve existing local metrics.
- Generated `runs/` outputs stay out of the repository.
