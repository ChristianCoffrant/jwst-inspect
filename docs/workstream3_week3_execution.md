# Workstream 3 Week 3 Execution Log

## Scope

Week 3 implements the Team 3 side of Gate 1: a local thin vertical slice that generates scripted rollout logs, computes coverage/standoff/safety/success metrics from those logs, emits a generated metrics table, and produces a first R2P report from matched rasterized and path-traced proxy episodes.

The Week 3 ship gate is:

> Gate 1 thin vertical slice passes for Team 3.

## Iteration 1: Generated Metrics Table

Implemented:

- `scripts/evaluate_thin_slice.py`
- generated rollout logs under `runs/thin_slice/`
- generated `metrics_table.csv`
- generated `metrics_report.json`

Decision:

- Metrics tables are generated only from rollout logs.
- Manual metric editing remains prohibited.

## Iteration 2: Coverage And Join Keys

Implemented:

- stable per-sample `coverage_patch`
- per-sample `episode_id`
- per-sample `frame_id`
- per-sample `target_region`
- join report using `episode_id + frame_id`

Decision:

- Coverage remains simple and patch-based for Week 3.
- Unsafe samples remain excluded from scored coverage.

## Iteration 3: R2P Placeholder

Implemented:

- matched rasterized proxy rollout
- matched path-traced high-glare proxy rollout
- generated `r2p_report.json`

Decision:

- Path-traced rows are proxy-labeled unless backed by synced Slurm OCI RTX logs.
- No official GPU result is accepted without synced logs and run-registry metadata.

## Week 3 Ship Gate

Pass criteria:

- scripted approach rollout runs from fixed config
- rasterized proxy and path-traced proxy evaluations are scored
- coverage, standoff, safety, success, and normalized score are computed from logs
- metrics table is generated by script
- R2P report is generated from logs
- Team 2 join keys are present and validated
- local metric rerun is deterministic

Validation commands:

```bash
python scripts/validate_contracts.py
python scripts/evaluate_thin_slice.py --config configs/experiments/thin_slice.yaml --output-dir runs/thin_slice
python -m unittest tests.test_thin_slice_evaluation tests.test_proxy_approach tests.test_metrics
```

## Slurm OCI Smoke Status

Week 3 does not claim an official GPU result. A future GPU result must have synced rollout logs, run-registry metadata, and locally regenerated metrics before it can be treated as official evidence.
